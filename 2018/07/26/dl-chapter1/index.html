1. Logical computations with Neurons

1.1 Artificial Neuron

It has one or more binary (on/off) inputs and one binary output, the artificial neuron simply activates its output when more than a certain number of its inputs are active.





2. The Perceptron

The Perceptron is one of the simplest ANN architectures which is based on a slightly different artificial neuron called a linear threshold unit(LTU). Then applies a step function to that sum and outputs the result:





Perceptron 是一个非常简单但结构已经很全面的神经网络了，它由多个linear threshold unit(LTU)组成， LTU是由一个线性计算加一个 Artificial Neuron组成（这里用的是Heaviside step）。

The most common step function used in Perceptron is the Heaviside step function:

$$ heaviside(z)= \begin{cases}
0, & \text {if $z <0$ } \newline
1, & \text{if $z \leq 0$ } \end{cases}$$

$$ sgn(z)= \begin{cases}
-1, & \text {if $z < 0$ } \newline
0, & \text {if $z = 0$ } \newline
+1, & \text{if $z > 0$ } \end{cases}$$

这个LTU的模型是不是看起来很熟悉？对了，跟之前上一章总结的Logistic Regression模型是不是如出一辙，唯一的区别就是sigmoid 换成了这里的Step函数。
所以说，后面用sigmoid 来代替这里Heaviside step函数也就是可以理解的了，原因呢，后面会给出。
这里引用之前一位大牛的话，简单的看逻辑回归就是一个完整的神经网络结构，一层，一个！

This Perceptron can classify instances simultaneously in to three different binary classes:





再来看Perceptron， 这里可以理解成由3个神经元（LTU）的一层神经网络结构，输出有3位，也就是可以做3分类任务。
接下来看一下Perceptron的训练方法。

2.1 Training

Hebb’s rule: the connection weight between two neurons is increased whenever they have the same output.

$$W_{i,j}^{(next_step)} = w_{i,j}+\eta(\hat{y_j}-y_j)x_i$$

wi,j is the weights between the ith neurons and jth neurons.

The decision boundary of each output neuron is linear, so Perceptrons are incapable of learning complex patterns(just like LR). It called the Perceptron convergence theorem.

  • 这个训练方法我觉得可以理解成梯度下降的简化版本，一样的结构类型，但是更简单的逻辑：同输出就增加，逆输出就减少。
  • 然而，这里提到了Perceptron一个很致命的问题，就是Perceptron看似很美好，结构清晰逻辑明了，但问题就是，它的适用范围只能处理线性问题。（这也很容易理解，毕竟就是只用了一层线性结构。）但带来的问题就是，它连XOR问题都解决不了，这样肯定和人们之前对它的期望值是非常不对等的，因此，对它抱有的热情也就慢慢无奈的退却了。
  • 当然，这个问题在Multi-Layer这个概念提出后，就随之消失了。

3. Multi-Layer Perceptron and Backpropagation

3.1 MLP

Stacking multiple Perceptrons could solve the XOR problem which is called a Multi-layer Perceptron(MLP):





An MLP is composed of one (passthrough) input layer, one or more layers of LTUs.When an ANN has 2 or more hidden layers, it is called a deep neural network(DNN).





简单的看可以是吧两个Perceptron叠起来，这里就是机器学习中stacking的概念（本来是会在基础机器学习最后一章中提出的）。这样就增加了模型的非线性，可以轻松的搞定XOR问题了。然后看一看，这个多层的Perceptron“s”要怎么训练呢？

3.2 Backpropagation

Backpropagation could be described as Gradient Descent using reverse-mode autodiff.

与之前传统的Perceptron相比，这里的训练方法就“升级”了。用了之前说的“GD”的概念。这个GD除了会作曲唱歌，还可以轻松搞定模型最优化的问题。但是，对于这个多层叠加的模型，就需要解决两个大方向上的问题：各层中导数如何求取和连续可导的问题。

Autodiff
a. Firstly, makes a prediction (forward pass), measures the error,
b. Then goes through each layer in reverse to measure the error contribution from each connection,
c. Finally, slightly tweaks the connection weights to reduce the error(Gradient Descent Step).

For providing the gradient they replaced the step function with the logistic function. Cause the original step function only contains the flat segments.

关于求取各层中导数的问题，这里用了一个reverse版本的autodiff的方法，也就是配合用了dual number的方法，这个后面我会单独写一小章介绍的，暂时可以理解为，正反算一遍，导数都清楚，不需要特别去算每层的导数表达式来算梯度。

Other activation functions:
The hyperbolic tangent function tanh(z):
S-shaped, continuous, and differentiable. But output value ranges from -1 to 1(instead of 0 to 1 in the case of the logistic function), which tends to make each layer’s output more or less normalized for speeding up convergence.
The ReLU function:
f(z) = max(0,z). It is continuous but not differentiable at z=0. However it works well and fast to compute.





The output layer is typically modified by replacing the individual activation functions by a shared softmax function.( Most for classification)

关于连续可导问题，这里如果使用原来的step函数显然没有意义的，毕竟这个梯度几乎就找不到，所以这里提供了很多中别的actiavtion函数，首先就是之前说的sigmoid，也包含tanh和比较火的Relu。当然下一章优化模型中，会提到更多版本的变种actiavtion函数，用于解决之前老版本中遇到的各种各样的问题。

4. Fine-Tuning Neural Network Hyperparameters

4.1 Number of hidden layers

  • Deep networks have a much higher parameter efficiency than shallow ones: they can model complex functions using exponentially fewer neurons than shallow nets, making them much faster to train.(Deep > Shallow)
  • Lower hidden layers model low-level structures(e.g., line segments of various shapes and orientations), intermediate hidden layers combine these low-level structures to model intermediate-level structures (e.g., squares, circles), and the highest hidden layers and the output layer combine these intermediate structures to model high-level structures(e.g., face).
  • Kickstart training by reusing the lower layers of the first network is possible and effective.
  • Start with just one or two hidden layers and it will work just fine. For complex problems, gradually ramp up the number of hidden layers, until you start overfitting the training set.
  • However reusing parts of a pretrained state-of-art network that performs a similar task. Training will be a lot faster and require much less data.

4.2 Number of Neurons per Hidden Layer

  • Still somewhat black art( as common, the form of a funnel or all the same);
  • A simpler approach is to pick a model with more layers and neurons than you actually need, then use early stopping to prevent it from overfitting. Just like “stretch pants” approach: just use large stretch pants that will shrink down to the right size.

4.3 Activation Functions

  • Using the ReLU activation function in the hidden layers;
  • For output layer Softmax is preferred or no activation function at all for regression tasks
